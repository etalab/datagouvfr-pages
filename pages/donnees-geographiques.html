Les données à composante géographique

Les données à composantes géographiques sont souvent indispensables pour réaliser des analyses. Sont référencées ici les principaux jeux de données disponibles sur data.gouv.fr . Celle-ci n'est pas exhaustive et est ouverte aux contributions .

Un certains nombre de ces données font office de référentiel qui servent de pivot avec d'autres jeux de données. Elles font parties du SPD (Service Public de la Donnée) . Voir aussi cet article sur comment lier des données à un référentiel

Données limites administratives

Données communales

Le Code Officiel Géographique (COG). C’est une donnée pivot qui permet de rattacher les données à d’autres jeux de données, à des données géographiques ou de faire des calculs à des niveaux administratifs issus d’une combinaison de communes.

  • Code officiel géographique avec les données sous forme JSON dans le cadre de l’API Découpage Administratif ( SPD )

Données IGN

  • Admin Express : données communales à partir de 2017 jusqu’à nos jours. Les versions récentes intègrent les EPCI
  • Geofla : données communales, cantonales et départementales en 1997, 2002 puis régulièrement de 2010 à 2016 régulièrement

La précision géométrique de ces limites a changé au fil du temps. Lors du passage Geofla vers Admin Express puis avec le passage de Admin Express en version 3. Voir cet article pour faire la distinction

Ces données font partie du SPD via le RGE (Référentiel à Grande Échelle)

Il faut noter que les Collectivités d’Outre-Mer (COM) ne sont pas présentes. Si vous voulez une version avec les COM, Etalab produit une version avec Admin Express pour métropole et DOM avec les géométries venant d’OpenStreetMap pour les COM. Voir les données générées par Etalab (avec le dépôt associé pour générer )

Pour ne pas avoir à gérer les DOM/TOM du fait de l’éloignement, lorsqu'on produit des cartes France entière, il existe une version de données avec les Contours des communes de France simplifié, avec régions et département d'outre-mer rapprochés

Données OpenStreetMap

Pour les données communales, celle-ci ont tracées depuis un fond raster communal de la direction générale des Finances publiques (DGFiP) comme cet article "Achèvement du tracé collaboratif des limites communales françaises dans OpenStreetMap" le retrace.

  • Extractions de OpenStreetMap France sur data.gouv.fr

Données postales

  • Base officielle des codes postaux de La Poste. Comme la donnée n'est qu'à date courante, il est possible d'avoir les mêmes données archivées plus anciennes sur OpenDatarchives
  • Fond de carte des codes postaux https://www.data.gouv.fr/fr/datasets/fond-de-carte-des-codes-postaux/

Données intracommunales et carroyées haute-résolution

  • IRIS: Il existe deux produits
    • Contour IRIS adapté pour représenter sur une carte car simplifiée
    • IRIS GE comme Grande Échelle adapté pour dénombrer dans quel IRIS un habitation est par exemple car la donnée a une plus grande précision
  • Bureaux de vote: pas de renseignement national. Au cas par cas, mis à disposition par la commune. Repris dans OpenStreetMap
  • Quartiers (cas par cas): soit disponible sur des portails OpenData soit sur OpenStreetMap
  • Données carroyées 200m
